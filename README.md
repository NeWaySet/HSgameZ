# Qt Battlegrounds

Карточная игра на `C++17 + Qt`, вдохновлённая режимом `Hearthstone: Battlegrounds`.  
Проект представляет собой прототип авто-баттлера с фазой покупки, таверной, героями, столом существ, автоматическим боем, локальной сетевой игрой и локальным режимом `сам с собой`.

## Назначение проекта

Цель проекта - реализовать играбельный настольный прототип карточной игры с графическим интерфейсом на `Qt`, в котором:

- есть два игрока;
- каждый игрок управляет героем с запасом здоровья;
- в фазе покупки игрок тратит монеты на покупку существ из таверны;
- купленные существа попадают в руку и затем выставляются на стол;
- после завершения подготовки запускается автоматический бой;
- поддерживается игра по локальной сети, через `Bluetooth` и на одном компьютере.

## Основные возможности

- герои с запасом здоровья;
- раундовая система;
- монеты, начинающиеся с `3` и увеличивающиеся каждый новый раунд;
- таверна на `7` карт с возможностью обновления за `1` монету;
- рука игрока;
- стол до `7` существ;
- покупка существ;
- выставление существ на стол;
- продажа существ со стола;
- ручная атака существом по существу противника;
- ручная атака существом по герою противника;
- автоматический бой между столами двух игроков;
- механика `Провокации`;
- возможность атаковать героя, если на столе противника нет существ с `Провокацией`;
- победа при снижении здоровья противника до `0` или ниже;
- возможность задать имена игроков через интерфейс;
- локальная игра по `LAN (TCP)`;
- опциональная игра по `Bluetooth`, если в сборке Qt доступен модуль `Qt Bluetooth`;
- локальный режим `сам с собой` на одном компьютере;
- скрытие информации между ходами в локальном режиме;
- затемняющая “ширма” для честной передачи устройства другому игроку;
- поддержка полноэкранного режима;
- корректный запуск в развёрнутом окне и улучшенная работа полного экрана на `Ubuntu`;
- более широкий адаптивный интерфейс под `3:2`, `16:9` и `16:10`.
- переработанный визуальный стиль: верхняя шапка, панельный layout и улучшенные карточные списки.

## Игровые механики

### Фаза покупки

В начале партии игрок получает `3` монеты.  
В каждом следующем раунде максимальное количество монет увеличивается, но не превышает `10`.

Во время фазы покупки игрок может:

- купить существо из таверны;
- обновить таверну за `1` монету;
- выставить купленное существо из руки на стол;
- продать существо со стола и вернуть `1` монету;
- завершить подготовку кнопкой `Готов к бою`.

### Таверна

Таверна содержит `7` случайных существ.  
Каждое существо имеет:

- имя;
- атаку;
- здоровье;
- стоимость;
- дополнительное свойство `Провокация` у части существ.

### Стол и рука

- рука может содержать до `10` карт;
- на столе может находиться до `7` существ;
- существа на столе участвуют в автоматическом бою;
- после завершения боя стол не сохраняет боевые ранения на следующий раунд.

### Автоматический бой

Во время боя:

- существа атакуют по очереди;
- цель выбирается автоматически;
- если у противника есть существо с `Провокацией`, атаковать можно только его;
- если `Провокации` нет, существо может случайно ударить либо по вражескому существу, либо напрямую по герою;
- бой продолжается до уничтожения одного из столов или до гибели героя.

Урон герою может наноситься двумя способами:

- напрямую во время боя, если существо атакует героя;
- после победы в раунде, когда оставшиеся в живых существа наносят дополнительный урон проигравшему герою.

### Ручные атаки игрока

Игрок может выбрать своё существо на столе и выполнить одно из действий:

- атаковать выбранное вражеское существо;
- атаковать вражеского героя.

Ограничения:

- одно существо может атаковать один раз за раунд;
- если на столе противника есть `Провокация`, сначала нужно атаковать именно такое существо;
- если здоровье героя противника становится `0` или меньше, партия завершается победой атакующей стороны.

## Режимы игры

### 1. Игра по сети

Поддерживается подключение двух игроков по локальной сети.

Доступны варианты:

- `LAN (TCP)` - основная сетевая игра через `QTcpServer` и `QTcpSocket`;
- `Bluetooth` - дополнительный транспорт при наличии модуля `Qt Bluetooth`.

Один игрок создаёт комнату, второй подключается по адресу.  
После подключения состояния игроков синхронизируются по сети, а результат боя рассылается обоим участникам.

### 2. Локальный режим `сам с собой`

Этот режим предназначен для игры двух игроков на одном компьютере.

В нём можно:

- начать локальный матч;
- играть по очереди за `Игрока 1` и `Игрока 2`;
- вручную передавать управление;
- скрывать текущую руку, таверну и стол активного игрока;
- использовать затемняющую ширму между ходами;
- запускать бой без сети.

## Интерфейс и адаптация

Интерфейс приложения переработан под более широкие форматы экрана и теперь лучше подходит для настольного запуска.

Что изменено:

- окно по умолчанию открывается развёрнутым;
- добавлена верхняя информационная шапка;
- основная рабочая зона разделена на боковую панель и широкую игровую область;
- интерфейс адаптирован под экраны с соотношением сторон `3:2`, `16:9` и `16:10`;
- полноэкранный режим доступен через кнопку и горячую клавишу `F11`;
- на `Ubuntu` выход из полноэкранного режима дополнительно поддерживается через `Esc` с возвратом в развёрнутое окно;
- обновлены цвета, панели, кнопки, журналы и списки карт для более выразительного внешнего вида.

## Управление

Основные действия в интерфейсе:

- `Создать комнату` - запуск сетевой игры в режиме хоста;
- `Подключиться` - подключение к уже созданной комнате;
- `Начать локальную игру` - запуск режима `сам с собой`;
- `Обновить таверну (-1)` - обновление списка карт в магазине;
- `Купить в руку` - покупка выбранной карты из таверны;
- `Выставить на стол` - перенос выбранной карты из руки на стол;
- `Продать со стола (+1)` - продажа выбранного существа;
- `Атаковать существо` - атака выбранным существом по выбранной цели на столе соперника;
- `Атаковать героя` - атака выбранным существом по герою соперника;
- `Готов к бою` - завершение подготовки;
- `Передать управление` - ручная передача хода в локальном режиме;
- `Показать ход` - открытие экрана следующему игроку.
- `Применить имена` - обновление имён игроков.
- `Полный экран` или `F11` - переключение полноэкранного режима.
- `Esc` - выход из полноэкранного режима.

## Структура проекта

Основные файлы проекта:

- `src/main.cpp` - точка входа приложения;
- `src/mainwindow.h` и `src/mainwindow.cpp` - графический интерфейс, режимы игры, локальный режим и связь UI с игровой логикой;
- `src/autobattler.h` и `src/autobattler.cpp` - игровая модель, карты, раунды, таверна, бой и расчёт результата;
- `src/networkmanager.h` и `src/networkmanager.cpp` - работа с `TCP` и `Bluetooth`;
- `CMakeLists.txt` - сборка проекта;
- `.gitignore` - исключения для артефактов сборки;
- `README.md` - описание проекта.

## Сборка проекта

Пример сборки через `CMake`:

```bash
cmake -S . -B build
cmake --build build
```

Для сборки необходимы:

- компилятор с поддержкой `C++17`;
- установленный `Qt` (`Qt5` или `Qt6` в зависимости от конфигурации проекта);
- `CMake`;
- модуль `Qt Bluetooth`, если нужна поддержка `Bluetooth`.

## Порядок игры

### Сетевая партия

1. Один игрок создаёт комнату.
2. Второй игрок подключается по `LAN` или `Bluetooth`.
3. Оба игрока покупают существ в таверне.
4. Купленные существа помещаются в руку.
5. Игроки выставляют существ на стол.
6. При необходимости обновляют таверну.
7. После завершения подготовки нажимают `Готов к бою`.
8. Когда готовы оба игрока, запускается автоматический бой.
9. После завершения боя начинается следующий раунд, если ни один герой не побеждён.

### Локальная партия

1. В поле режима выбрать `Сам с собой`.
2. Нажать `Начать локальную игру`.
3. Собрать стол для текущего игрока.
4. Нажать `Готов к бою` или `Передать управление`.
5. Передать устройство следующему игроку.
6. Следующий игрок открывает экран кнопкой `Показать ход`.
7. После готовности обеих сторон запускается локальный бой без сети.

## Особенности реализации

- проект написан в объектно-ориентированном стиле;
- игровая логика отделена от сетевой части;
- состояние игрока поддерживается как для сетевого режима, так и для режима `сам с собой`;
- свойства карт сериализуются в `JSON` для обмена между устройствами;
- интерфейс оформлен в стилистике, вдохновлённой `Hearthstone`;
- layout оптимизирован под широкие десктопные экраны и Linux-сценарий запуска в полноэкранном режиме.

## Ограничения текущего прототипа

- это не полная копия `Hearthstone Battlegrounds`, а учебный и демонстрационный прототип;
- в проекте пока нет продвинутых эффектов вроде `Боевого клича`, `Предсмертного хрипа`, `Божественного щита`, уровней таверны и синергий племён;
- после боя существа не сохраняют постоянные повреждения между раундами;
- внешний вид только стилизован под `Hearthstone`, но не повторяет оригинальный интерфейс полностью.
